- 2.1.1 -

demo.hello.Hello1

What about using a parameter sender just to indicate who is sending the hello message ? This can be done by adding an input signal sender to the module definition :

Hello1.lilas :
<?xml version="1.0" encoding="UTF-8"?> <module> <signal mode="in" name="sender" sensitive="true" type="java.lang.String"/> <code language="java"/> </module>

Here, sensitive="true" means that the module is activate when a value is modified on the signal. As we did for module Hello, whe modify the activation method of the generated java file demo/hello/Hello1.java :

Hello1.java :
package demo.hello; public class Hello1 extends Hello1_LILAS { public Hello1(String nomDansModuleParent, lilas.base.Module parent, lilas.base.Signal<java.lang.String> sender_signal) throws lilas.base.ExceptionLilas { super(nomDansModuleParent, parent, sender_signal); } @Override final protected void activation() throws lilas.base.ExceptionLilas { System.out.println("Hello world from "+sender+" !"); } }

Now, launching this module with a value of input signal give :

$ lilas /demo/hello/Hello1.lilas sender=myself Lilas ready ! Hello world from myself ! Lilas completed ! $
