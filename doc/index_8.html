- 2.1 -

demo.hello.Hello

We define the demo.hello.Hello module with the simplest LILAS description file LilasProject/demo/hello/Hello.lilas:

Hello.lilas :
<?xml version="1.0" encoding="UTF-8"?> <module> <code language="java"/> </module>

When launching this module the first time, LILAS create the file Hello.java in the same directory as Hello.lilas :

Hello.java :
package demo.hello; public class Hello extends Hello_LILAS { public Hello(String nomDansModuleParent, lilas.base.Module parent) throws lilas.base.ExceptionLilas { super(nomDansModuleParent, parent); } @Override final protected void activation() throws lilas.base.ExceptionLilas { } }
The activation method is the method executed by LILAS each time the LILAS scheduler activate the module. So, we have to modify the method with :
@Override final protected void activation() throws lilas.base.ExceptionLilas { System.out.println("Hello world !"); }

Now, launching again this module give :

$ lilas /demo/hello/Hello.lilas Lilas ready ! Hello world ! Lilas completed ! $

That's all ! Now, let see some variations to introduce a few elements of LILAS.
