- 5.5.1 -

logic

Page ParserLogic.html provides a complete grammar of the language produced by the jjdoc tool. The language used to describe a circuit in the form of a list of instructions. A statement begins with a connection identifier or a list of connection identifiers, the character = then a logical expression and, finally, character ;. Comments can be added at any time between // and the end of line (like the comments of java language).

A connection identifier may indicate a connection size (number of bits) by attaching to the character identifier : followed by the value of size. If size is not specified, it will be synthesized by the LILAS system.

The following sections provide some examples of LC language instructions.

A literal corresponding to a value of lilas.type.StdLogic type can have one of the following formats:

  • 0[b,B]([0-1,u,U,z,Z,x,X,w,W,l,L,h,H,-]):([0-9])+ : value of one bit repeated on all bits
  • 0[b,B]([0-1,u,U,z,Z,x,X,w,W,l,L,h,H,-])* : binary format
  • 0[x,X]([0-9,a-f,A-F,u,U,z,Z,x,X,w,W,l,L,h,H,-])+(:([0-9])+)? : hexadecimal format
  • ([0-9])+:([0-9])+ : decimal format
Optionally, the literal may contain one or more characters '_' not material. When the number of bits in size is not given (binary or hexadecimal format), the number of bits of the value is the number of binary characters or four times the number of hexadecimal characters.

Examples :

"0bx" : 1 bit value "0bx:8" : 8 bits value equivalent to 0bxxxxxxxx "3:5" : 5 bits value equivalent to "0b00011" "3:5" : 5 bits value equivalent to "0b00011" "0x3:5": 5 bits value equivalent to "0b00011" "0x3": 4 bits value equivalent to "0b0011" "0b00_1001_1100": 10 bits value equivalent to "0b0010011100"

5.5.1.1 Logical expression

The logical operators available to constitute a logical expression are and, or, exclusive-or not:

  • and : And
  • or : Or
  • xor : Xor
  • not : Not

These operators are applied to explicit connections or the output logic sub-expressions or logic functions such as mining, smelting, multiplexing or the constant generation logic.

5.5.1.2 Arithmetic expression

The arithmetic operators available to constitute a logical expression are plus, minus and mult:

  • plus : Addition
  • minus : Subtraction
  • mult : Multiplication

These operators are applied to explicit connections or the output logic sub-expressions or logic functions. The arguments (there are 2) must be the same size in the case of addition or subtraction (the result will be the same size), or of different size in the case of multiplcation (the size of the result will be then the sum of the sizes of arguments).

Example: an adder circuit can be described with the following code:

Add4b.lilas :
<?xml version="1.0" encoding="UTF-8"?> <module> <signal mode="out" name="q" type="lilas.type.StdLogic" size="4"/> <signal mode="in" name="clk" type="lilas.type.StdLogic" sensitive="true" init="0b0"/> <signal mode="in" name="d1" type="lilas.type.StdLogic" size="4"/> <signal mode="in" name="d2" type="lilas.type.StdLogic" size="4"/> <code language="logic"><![CDATA[ q <= d1 plus d2; ]]></code> </module>

5.5.1.11 Moore machine

The syntax of the beginning of a definition of a Moore machine is:

moore smclock=nomConnexionClock smreset=nomConnexionReset sminit=nomEtatAuReset state=nomConnexionNomEtatCourant out= (nomSignalLogique1, ...) { nomEtat1 (nomSignal <= val, ...) on expLog1 target nomEtat; on.. } { .. }
