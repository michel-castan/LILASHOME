Quick start tutorial

We present 3 examples of lilas modules :

  • the traditional hello program, a module that simply print hello world,
  • a simple computation of sqare root in order to present differents tools of LILAS,
  • a counter program as an example of FPGA application with LILAS.

These modules are included in the java archive LILASHOME/lib/lilasV4demo.jar.

In order to launch a LILAS module, you have to use the lilas script file created in your working directory (for example LilasProject) or directly the command java -jar pathToLILASHOME/LILASHOME/lib/lilasV4.jar from one directory under your working directory. In the rest of this document, we will use the command lilas as an alias of this launching command.

The LILAS module pkg1.pkg2. ... .pkgn.ModuleName is described by a xml file : ModuleName.lilas The prefix pkg1.pkg2. ... .pkgn is the package name of the module. As a package for a java class, a package name correspond to a directory hierarchie in a file system. Thus, the path of this xml file is /pkg1/pkg2/.../pkgn/ModuleName.lilas; the first "/" character indicate one of the directory of the java classpath indicated by the LILAS configuration file.

When we refer to a LILAS module in a launching command, we can use an absolute path or a path relative to the current directory. For example, these commands can be use to launch the LILAS module pkg1.pkg2. ... .pkgn.ModuleName (we assume here the working directory as the current directory of the first command, cd is the unix command to change the current directory) :

lilas /pkg1/pkg2/.../pkgn/ModuleName.lilas lilas pkg1/pkg2/.../pkgn/ModuleName.lilas cd pkg1 lilas /pkg1/pkg2/.../pkgn/ModuleName.lilas lilas pkg2/.../pkgn/ModuleName.lilas cd pkg2a lilas /pkg1/pkg2/.../pkgn/ModuleName.lilas lilas ../pkg2/.../pkgn/ModuleName.lilas

Because of the java classpath, many files can be candidate for a given module name. In that case, only the first one found (considering the order indicated by the classpath) will be considered.

Lastly, if we launch lilas without indicating a file, we will have a window showing a tree hierarchical view of all LILAS modules from the current classpath. Package demo is given by java archive LILASHOME/lib/lilasV4demo.jar, while package lilas is given by java archive LILASHOME/lib/lilasV4.jar. Package demo modules can be used directly from the java archive, however, to modify one of these modules, we have to get a copy in LilasProject directory; this can be done with the option -extract of the LILAS command :
lilas /pkg1/pkg2/.../pkgn/ModuleName.lilas -extract

When using this LILAS Main Frame, we have to complete the LILAS command at the bottom of the window and then either enter or click on the Go button to start the command execution.
