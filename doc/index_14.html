- 2.2.3 -

Conditional configuration

So far, we have defined two ways to compute Sqrt function. We can now define a parameterizable module demo.srqt.Sqrt which will use either the first version or the second version depending the value of a parameter. The LILAS definition of this module will be :

Sqrt.lilas :
<?xml version="1.0" encoding="UTF-8"?> <module> <doc>Square Roots by Newton's Method, conditional configuration</doc> <signal mode="constant" name="version" type="java.lang.Integer"/> <signal mode="in" name="x" sensitive="true" type="java.lang.Double" init="0.0"/> <signal mode="out" name="y" type="java.lang.Double" init="0.0"/> <case> <when cond="version==1"> <Sqrt1 x="x" y="y"/> </when> <otherwise> <Sqrt2 x="x" y="y"/> </otherwise> </case> </module>

Keep in mind that the version value is taken into account when creating the instance of SQRT. That is why the version signal is defined as constant. In addition, having no initial defined value, the latter must be given at the time of the module instanciation.

For example, command lilas /demo/sqrt/Sqrt.lilas -awt x=2.0 version=1 give

while command lilas /demo/sqrt/Sqrt.lilas -awt x=2.0 version=2 give

As we can see, the module demo.sqrt.Sqrt use an intermediate module demo.sqrt.Sqrt_0 or demo.sqrt.Sqrt_1 depending on version value, but this is transparent to LILAS user.
