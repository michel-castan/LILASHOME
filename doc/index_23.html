lilas.base.Signal

A signal is a connection to convey a value of a given type. The signal is defined by an XML element signal. It is defined with attributes, some common to all signals and some specific to the type of associated data.

List of common attributes :

  • name : the local name of the signal in the module.
  • type : the type of the data carried by the signal
  • sensitive : true indicates that the module is sensitive to this signal (false by default). In the case of logic module (which can be converted into VHDL), the sensitivity is changed according to the logic defined in the code module (a warning is printed to signal an automatic modification on a signal whose sensitivity is not announced according to logic description).
  • mode : the possible values ​​are
    • in : the signal can only be read in the module
    • out : the signal can only be write from the module
    • inout : the signal can be read or written by the module
    • constant : the initialization value of the signal is final upon instantiation. Such a signal is replaced by a constant value. If the value is provided outside the module, such signal becomes a generic variable.
    • internal : the signal is generated internally in the module. An internal signal (or constant) is analogous to local variables in a procedure while a signal in or out or inout is analogous to a parameter procedure.
    • dummyIntern : the internal signal is used by a graphical user interface (or a script) internal to the module.
    • dummyExtern : the external signal is only used by a graphical user interface (or a script) internal to the module.
  • init : the initialization value of the signal when it is created (for internal signals). The default initialization value is the default value of the associated type.
  • optional : true indicates that the signal may be absent in a module call (for external signals), false (the default) indicates that the presence of this signal is mandatory.
  • position : { LEFT, RIGHT, UP, DOWN } indicates the position of the external signal (without effect an internal signal) in the automatic svg representation of a module instance. When the position is not specified, the default is LEFT for IN mode, RIGHT for OUT mode, DOWN for INOUT mode and UP for CONSTANT and DUMMYEXTERN modes.

Example of specific attributes :

  • size : for signals associated to logical values, this attribute specifies the number of bits of the logic value.

At any signal associated XML declaration is an instance of the class java lilas.xml.DefSignal.

5.1.1.1 Specifying a type class

Any java object can be transmit through a signal. However, it must ensure that the class Signal<T> can generate instances of the associated class T, which can be done by calling the constructor of T set with the signature public T(Stringstr). So, this constructor must be available.

If several values ​​have to be given, they should be encoded in a string in a way or another. For example, if we wish to manipulate a two dimensions table of Double defined by:

java.lang.Double[][] table = new java.lang.Double[10][20];
Nous pourrons le faire en définissant la classe TableDouble par :
public class TableDouble { Double[][] table; public TableDouble(String init) { String[] param = init.split(":"); table = new Double[Integer.parseInt(param[0])][Integer.parseInt(param[1])]; } ... }

the signal declaration will then be

<signal name="table" type="TableDouble" init="10:20"/>
