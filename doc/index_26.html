- 5.2 -

lilas.awt

The "awt" element of a module allow to specify a GUI from a number of elements described in the following subsections. The "awt" element has a required attribute "name" for the identifer and an optional attribute "layout" described below in the panel element.

One of the elements constituting the awt element may be an instance of awt. Simply use an element name "instance" with the attribute "awt" used to designate the instance (if the instance is not defined in the module, the awt attribute indicates the path from the root SystemL instance) :

<awt name="awt1" layout="grid" colonnes="1"> <Text name="txtX1" signal="x" taille="10" label="x :" editable="true"/> <Text name="txtY2" signal="y" taille="10" label="y :" editable="false"/> </awt> <awt name="awt2" layout="grid" colonnes="1"> <instance name="instaw1" awt="awt1"/> <instance name="instaw1bis" awt="var1.var2.awt1"/><!-- awt1 defined in module instance referenced by the variable var1.var2 --> </awt>

It is possible to refer to an element created by a particular Java method; just set an instance awt without content and same name as the java method :

ComponentAWT.lilas :
<?xml version="1.0" encoding="UTF-8"?> <module> <awt name="myComponent"/> <code language="java"/> </module>
with the instanciation java class :
ComponentAWT.java :
package test.lilas.awt; import javax.swing.JComponent; public class ComponentAWT extends ComponentAWT_LILAS { public ComponentAWT(String nameInParentModule, lilas.base.Module parent) throws lilas.base.ExceptionLilas { super(nameInParentModule, parent); } @Override final protected void activation() throws lilas.base.ExceptionLilas { } @Override public JComponent myComponent() { JComponent comp = new javax.swing.JSlider(javax.swing.JSlider.HORIZONTAL, 0, 100, 50);; return comp; } }
