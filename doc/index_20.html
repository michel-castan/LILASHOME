Configuration file

The configuration file is an XML document used to change various LILAS settings. The path to the configuration file can be passed to LILAS system with the help of the -config options of the command line :

$ lilas -config=/xxx/yyy/name.ext ...
and so, the file can be anywhere in the file system and can have any name. However, if the name of this file is lilasConfig.xml, LILAS can detect automatically by searching in current directory or parent directories. In that case, we do not need to use the -config options of the command line. If the configuration file is not found, LILAS proposed to create one automatically upon startup.

The root element of an XML configuration file is config. It may contain the following elements (in any order) :

  • <jar path="xxx"/> : where xxx is the path of a jar file or the path of a directory (in that case, we consider all the jar files containing in that directory)
  • <dir path="xxx"/> : where xxx is the path of a directory
  • <var name="xxx" value="yyy"/> : where xxx is a name of lilas parameters (static field of lilas.Parameter class), and yyy the value of this parameter.

The jar elements and dir elements are used to build the java classpath for lilas with two specials parameters dirSrc and dirBin. The classpath contains first the source directory of the LILAS project, indicated by dirSrc variable, following by the binary directory of the LILAS project, indicated by dirBin variable, following by the jar files associated to jar element or the directory associated to the direlement, in order of appearance in the config file. If the variable dirSrc is not set, LILAS consider the current directory; if the dirBin is not set, LILAS consider the value associated to dirSrc variable.

The different variables that can be changed by a var element are those already presented as option in the section on LILAS launching command : verbose, logLevel, fileLog, javaLibraryPath, dirSrc, dirBin, withHistoric, parallelMode, board, nexys3LoadCmd, ms6LoadCmd, ftdiJava, stepByStepMode, activationTimeDefault, simulatedTimeMode, elementaryGateDelay, bitValueDefault, STD_LOGIC_1164_modified_Mode.

For example, here is the configuration file that can be created automatically by LILAS :

<?xml version="1.0" encoding="UTF-8"?> <config> <jar path="~/LILASHOME/lib/"/> <var name="javaLibraryPath" value="~/LILASHOME/lib"/> <var name="dirSrc" value="~/LilasProject"/> <var name="logLevel" value="INFO"/> </config>
